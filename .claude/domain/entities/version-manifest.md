# Version Manifest

**Context**: versioning | **Type**: aggregate-root

## Definition
Root data structure containing all API versions and their metadata. Serves as the entry point for all version-related data in the application.

## Type
```typescript
// src/shared/types/version.ts
type VersionManifest = {
  serviceName: string;
  generatedAt: string;
  totalVersions: number;
  versions: VersionEntry[];
  availableDiffPairs: string[];  // "v0.8.0_v0.9.0" format
};
```

## Source
- **Build-time**: Generated by `scripts/generate-versions-index.mjs`
- **Runtime**: Fetched from `/data/versions.json` via `fetchVersions()` in `shared/lib/api.ts`
- **Hook**: `useVersions()` — returns `AsyncState<VersionManifest>`

## Usage
- Dashboard: stats, timeline, recent changes
- Diff page: version picker, available diff pairs
- Changelog page: version list with diffs
- Reference page: version selector dropdown

## Business Rules
- `availableDiffPairs` controls which from/to combinations are valid in the version picker
- Versions are sorted chronologically (by semver)
- `generatedAt` reflects the last build time
