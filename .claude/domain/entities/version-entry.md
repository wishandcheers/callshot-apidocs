# Version Entry

**Context**: versioning | **Type**: entity

## Definition
Single API version with per-group endpoint/path stats and an optional diff summary against the previous version.

## Type
```typescript
// src/shared/types/version.ts
type VersionEntry = {
  version: string;         // "v0.9.1"
  specVersion: string;     // OpenAPI info.version
  releasedAt: string | null;
  stats: VersionStats;
  diff: {
    previousVersion: string;
    summary: DiffSummary;
  } | null;                // null for earliest version
};

type VersionStats = {
  api: { pathCount: number; endpointCount: number };
  admin: { pathCount: number; endpointCount: number };
  schemaCount: number;
  tagCount: number;
};
```

## Source
- **Build-time**: Generated by `scripts/generate-version-meta.mjs` (individual meta) + `scripts/generate-versions-index.mjs` (assembled into manifest)
- **Raw specs**: `specs/:version/apiDocs-api.json` and `apiDocs-admin.json`
- **Service metadata**: `specs/:version/service-metadata.json` (optional, provides `releasedAt`)

## Business Rules
- Stats are split by spec group (`api` vs `admin`) for path and endpoint counts
- `schemaCount` and `tagCount` come from the API spec only
- `diff` is null for the first version (no previous to compare against)
- Endpoint count = sum of HTTP methods across all paths
