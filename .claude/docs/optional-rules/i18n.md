# Internationalization (i18n) Patterns

## When to Use
- Projects requiring multi-language support
- Global services

## Recommended Stack
- **react-i18next** + **i18next** (React)
- **vue-i18n** (Vue)
- **@formatjs/intl** (Framework-agnostic)

## Configuration (react-i18next example)

### Setup
```typescript
// shared/config/i18n.ts
import i18n from 'i18next';
import { initReactI18next } from 'react-i18next';
import Backend from 'i18next-http-backend';
import LanguageDetector from 'i18next-browser-languagedetector';

i18n
  .use(Backend)
  .use(LanguageDetector)
  .use(initReactI18next)
  .init({
    fallbackLng: 'ko',
    supportedLngs: ['ko', 'en', 'ja'],
    defaultNS: 'common',
    interpolation: {
      escapeValue: false, // React already escapes
    },
    backend: {
      loadPath: '/locales/{{lng}}/{{ns}}.json',
    },
  });

export default i18n;
```

## Translation File Structure

```
public/locales/
├── ko/
│   ├── common.json        # Common translations
│   ├── auth.json           # Authentication
│   └── validation.json     # Validation messages
├── en/
│   ├── common.json
│   ├── auth.json
│   └── validation.json
└── ja/
    └── ...
```

### Namespace per Feature
```json
// public/locales/ko/auth.json
{
  "login": {
    "title": "Login",
    "email": "Email",
    "password": "Password",
    "submit": "Sign In",
    "forgotPassword": "Forgot Password"
  },
  "register": {
    "title": "Sign Up"
  }
}
```

## Usage Patterns

### Component Usage
```typescript
import { useTranslation } from 'react-i18next';

function LoginForm() {
  const { t } = useTranslation('auth');

  return (
    <form>
      <h1>{t('login.title')}</h1>
      <Input label={t('login.email')} />
      <Input label={t('login.password')} />
      <Button>{t('login.submit')}</Button>
    </form>
  );
}
```

### Interpolation
```json
{ "welcome": "Hello, {{name}}" }
```
```typescript
t('welcome', { name: user.name })
```

### Pluralization
```json
{
  "items_one": "{{count}} item",
  "items_other": "{{count}} items"
}
```

## FSD Integration

```
features/auth/
├── ui/LoginForm.tsx     # uses useTranslation('auth')
└── ...

shared/ui/atoms/Button/
├── Button.tsx           # receives translated text via children
└── ...                  # shared/ui must NOT call t() directly
```

### Rules
- shared/ui components: receive translated text via props (no direct t() calls)
- features/entities: use their respective namespace
- No hardcoded user-facing strings

## Best Practices

### DO:
- Split namespaces by feature
- Use ICU MessageFormat for pluralization
- Include context in key names (`login.submit`, not just `submit`)
- Use `Trans` component for translations containing HTML

### DON'T:
- Concatenate translations (`t('hello') + name`)
- Generate dynamic keys (`t(\`status.${status}\`)` — use mapping instead)
- Call t() directly in shared/ui
